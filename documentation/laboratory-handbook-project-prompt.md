# Laboratory Handbook Project Prompt

## Project

Create:

# Volume 0 — Laboratory Operations Handbook

For:

Jamieson Group  
School of Chemistry  
University of Glasgow


The handbook complements:

# Volume 1 — Woody Computational Chemistry Workstation Guide


# Purpose

Create the definitive practical operating manual for the laboratory.

This is:

- a reference manual
- a training aid
- a practical guide


This is not:

- a textbook
- a theory document
- a collection of lengthy SOPs


# Audience

Users:

- undergraduate researchers
- PhD students
- postdoctoral researchers


# Laboratory Context

The group focuses primarily on:

- peptide chemistry
- chemical biology


Approximate group size:

6–10 members


Equipment includes:

- Buchi rotary evaporator
- chiller
- Isolera purification system
- microwave synthesiser
- analytical HPLC
- Syro II peptide synthesiser
- Alstra peptide synthesiser
- LCMS
- preparative HPLC
- semi-preparative HPLC
- NanoDrop
- freeze dryer


# Style

The handbook should feel like:

A professional scientific instrument manual combined with guidance from an experienced researcher.


Requirements:

- concise
- practical
- visually clean
- easy to navigate
- beginner friendly


# Development Workflow

Every new topic follows this process.


## Step 1 — Review Existing Information

Use:

- existing SOPs
- manuals
- notes
- previous documentation
- images
- user information


Treat existing material as the starting point.


## Step 2 — Interview

Ask questions.

Extract:

- practical knowledge
- common errors
- troubleshooting
- best practice


Never assume missing information.


## Step 3 — Documentation

Create the chapter.


## Step 4 — Review

Check:

- accuracy
- completeness
- consistency


## Step 5 — HTML

Generate HTML matching the handbook design.


# Chapter Structure

Where appropriate include:

- Quick summary
- Purpose
- Principle/background
- When to use
- When not to use
- Safety
- Preparation
- Required materials
- Procedure
- Cleaning
- Shutdown
- Maintenance
- Common mistakes
- Troubleshooting
- FAQs
- Related procedures


# Website

Use:

- HTML
- CSS
- JavaScript


No frameworks.

The handbook should operate as one website.

Individual chapters should remain separate files.


# Version Control

Use GitHub.

Each chapter should include:

- version number
- date
- change history


Maintain:

- changelog
- repository history


# Design

Woody guide is the design inspiration.

Maintain:

- typography
- navigation style
- callout style
- overall professionalism


However:

The design may evolve if improvements are identified.
